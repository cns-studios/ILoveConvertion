5. How We Process Your Data

5.1 File Processing Pipeline

  1. Upload & Encryption: File is encrypted immediately upon arrival using AES-256-GCM. Unique encryption key is derived for each job using HKDF-SHA256. Encrypted file is stored in /storage/inputs
  2. Processing: Job is queued in Redis (stores only Job ID, no file content). Worker creates temporary workspace in RAM-disk (tmpfs). File is decrypted into RAM for processing. Conversion/compression is performed using: libvips, ffmpeg, Ghostscript, QPDF, or Rembg AI service. Output is re-encrypted before storage.
  3. Delivery: Encrypted output is stored in /storage/outputs. You can download the processed file via unique download link. Job metadata is stored in PostgreSQL (Job ID, timestamps, file sizes, status).
  4. Deletion: Automated cleanup task runs every 15 minutes. All jobs older than 24 hours are permanently deleted. Includes: encrypted input/output files, database records, Redis queue entries.

5.2 AI Background Removal

When you use the background removal feature:

  • Your image is transmitted to our internal Rembg microservice (Python/Silueta model)
  • This service runs on the same server infrastructure
  • The image is processed in memory and immediately discarded
  • No external AI services are used
